# Lab 5 + Lab 6 Docker Master Guide (Learn + Viva + Teach)

This guide is written for your level: practical, no fluff, and exactly what to say/do in viva.

---

## 0) What is Docker (in your words)

Use this one-liner in viva:

> Docker runs your app inside an isolated container with all dependencies, so it behaves same on every machine.

### Key terms

- **Image**: Template (like a class in OOP).
- **Container**: Running instance of image (like an object in OOP).
- **Dockerfile**: Recipe to build image.
- **docker compose**: Runs multiple containers together (client + server).

## 2) Lab 5 (RPC in C) — what to explain

## Architecture

- `server/` has RPC server implementation.
- `client/` has RPC client caller.
- `docker-compose.yml` starts both on same Docker network.

## Run commands (single machine demo)

```bash
cd /Users/naish/Desktop/College/Sem6/DC/Labs/Lab5
docker compose up -d --build rpc-server
docker compose run --rm -it rpc-client ./run_client.sh rpc-server
```

## Stop commands

```bash
docker compose down
```

## Full cleanup (optional)

```bash
docker compose down --remove-orphans
docker image rm lab5-rpc-server lab5-rpc-client 2>/dev/null || true
```

## Viva points for Lab 5

- `rpcgen` generates client stub + server skeleton.
- `rpcbind` maps RPC program number to dynamic port.
- XDR ensures data portability across machines.
- Client and server are in separate containers to simulate two devices.

## 4) Two-device setup (important)

Yes, both labs can run across two devices, but do this carefully.

## Lab 5 (SunRPC) caveat

SunRPC may use dynamic ports; across different networks/firewalls this can be tricky.
Best for demo: same machine with compose.

## Lab 6 (RMI) across devices

Server machine:

```bash
cd server
docker build -t lab6-rmi-server .
docker run --rm -it --name lab6-rmi-server \
  -e RMI_HOSTNAME=<SERVER_DEVICE_IP> \
  -p 1099:1099 -p 2001:2001 \
  lab6-rmi-server
```

Client machine:

```bash
cd client
docker build -t lab6-rmi-client .
docker run --rm -it --name lab6-rmi-client \
  lab6-rmi-client ./run_client.sh <SERVER_DEVICE_IP> sample.txt
```

To find server IP:

```bash
ipconfig # Windows
ifconfig getifaddr en0 # or en1 Linux/Mac
```

Firewall/router must allow TCP `1099` and `2001`.

## 5) Most common errors + fixes

## Error: `container name already in use`

```bash
docker rm -f <container-name>
```

## Error: port already allocated (`1099`, `2001`, etc.)

You already have server running. Stop old one:

```bash
docker compose down
# or
# docker rm -f <old-container>
```

## Client hangs / no output

- Check server is actually running:

```bash
docker compose ps
docker compose logs rmi-server
```

- Ensure correct server host is used (`rmi-server` in compose, or server IP in cross-device mode).
