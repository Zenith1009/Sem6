### Topic 1: Network Basics, Types & Representations
This section forms the foundation. Expect 2-mark definitions or a 5-mark question asking you to classify network types with examples.

**1. Structured Notes & Important Terminology**
*   **Network (Graph):** An ordered pair $G(V, E)$, where $V$ is a set of nodes (vertices/entities) and $E$ is a set of edges (links/relations).
*   **Unipartite Network:** Links can join *any* pair of nodes without restriction (e.g., LAN network).
*   **Bipartite Network:** Vertex set $V$ is divided into two disjoint sets ($V_1, V_2$). Edges only connect a node in $V_1$ to a node in $V_2$. (Generalization: *n-partite networks*).
*   **Signed Network:** A network with a function $f: E \rightarrow \{+, -\}$ assigning positive (friend) or negative (foe) signs to edges. Crucial for "balance and status theory".
*   **Homogeneous Network:** All nodes and edges are of the *same type*.
*   **Multidimensional Network:** A multilayer network where each layer represents a different *type* of relationship (e.g., Layer 1: Social interaction, Layer 2: Car feature preference).
*   **Temporal Network:** Edges are represented as a 3-tuple $e_{ij} = \{v_i, v_j, t_{ij}\}$, where $t_{ij}$ indicates the duration/timestamp. Think of it as "snapshots" of a network over time.
*   **Hypergraph:** Edges (called hyperedges) can connect *multiple* nodes simultaneously (e.g., nodes = authors, hyperedge = a research paper connecting all co-authors).

**2. High-Weightage Concepts**
*   Differentiating between *Multidimensional* networks and *Hypergraphs*.
*   Network Representation: **Adjacency Matrix** (Square matrix $|V| \times |V|$ where $a_{ij}$ indicates an edge/weight) vs **Adjacency List** (Collection of lists showing neighbors for each node).

**3. Diagrams in Exam**
*   **Bipartite:** Draw two distinct columns of nodes. Draw lines only *across* the columns, never between nodes in the same column.
*   **Hypergraph:** Draw nodes as dots. Instead of drawing lines, draw circular "blobs" (like Venn diagram circles) that enclose 3-4 nodes together, and label it "Hyperedge".

**4. Common Mistakes**
*   Forgetting that in an Adjacency Matrix, **Self-loops** (a node connected to itself) are represented on the *main diagonal*. 
*   Confusing *Node-attributed* networks (where nodes have features like "Academic Affiliation") with *Multidimensional* networks.

**5. Memory Trick / Mnemonic**
*   For Network Types: **"U Be Studying Hard, My Team!"** -> **U**nipartite, **B**ipartite, **S**igned, **H**omogeneous, **M**ultidimensional, **T**emporal, **H**ypergraph.

**6. Practical Applications**
*   **Bipartite:** E-commerce (Users $\leftrightarrow$ Products).
*   **Healthcare:** Combating epidemics like SARS (2003 Taiwan Contact Network) and tracking super-spreaders.

### Topic 2: Levels of Social Network Analysis
Universities love asking, "Explain the different levels of SNA." This is a guaranteed 4-to-6 marker.

**1. Structured Notes & Important Terminology**
*   **Microscopic Level:** Analyzes pairs or small groups. Includes *Dyadic* (2 nodes), *Triadic* (3 nodes), and *Ego-centric circles* (a focal 'ego' node and its alters).
*   **Mesoscopic Level:** An intermediary level dealing with subsets of the population.
    *   *Communities:* Formed by frequent interactions among homogeneous nodes.
    *   *Network Motifs:* Subgraphs that repeat themselves frequently (highly effective for functional properties).
*   **Macroscopic Level:** Deals with the *entire network as a whole* to understand overall properties like Density, Diameter, and Degree Distribution.

**2. High-Weightage Concepts**
*   Clearly listing the specific metrics that fall under each level (e.g., Degree is Micro, Communities are Meso, Diameter is Macro).

**3. Diagrams in Exam**
*   Draw 3 separate boxes. Box 1 (Micro): Two nodes connected. Box 2 (Meso): A tightly-knit triangle or square of nodes (label: Motif). Box 3 (Macro): A large cloud representing a huge graph.

**4. Common Mistakes**
*   Students often classify "Network Motifs" under the Microscopic level because they are small. **Correction:** Motifs belong to the *Mesoscopic* level because they analyze repeating subgraphs across the population.

**5. Memory Trick / Mnemonic**
*   **3M:** 
    *   **M**icro = **M**e & You (Dyads/Egos)
    *   **M**eso = **M**y Group (Communities/Motifs)
    *   **M**acro = **M**asses (Entire graph/Diameter)

### Topic 3: Basic Network Measures & Proofs
This chapter is full of numericals and proofs. Write mathematical formulas clearly to grab full step-marks.

**1. Structured Notes & Important Terminology**
*   **Degree:** For undirected networks, it's the number of edges incident on node $v$. In directed networks, it splits into Indegree and Outdegree.
*   **Density ($\rho(G)$):** Ratio of actual edges to total possible edges. 
    *   Formula (Undirected): $\rho(G) = \frac{2 \times |E|}{|V| \times (|V| - 1)}$.
*   **Distance:** The length of the *shortest path* between two nodes.
*   **Diameter:** The *maximum distance* between *any* pair of nodes in the network.
*   **Degree Distribution ($P_k$):** Probability that a randomly chosen node has degree $k$. Formula: $P_k = \frac{N_k}{N}$ ($N_k$ is the number of nodes with degree $k$).
*   **Connectedness:** A directed graph is *Strongly Connected* if a directed path exists between *every* pair. It is *Weakly Connected* if replacing directed edges with undirected ones makes it connected.

**2. High-Weightage Concepts (Guaranteed Proofs!)**
*   **Theorem 1:** Sum of degrees in an undirected graph = $2 \times |E|$. (*Proof Idea:* Every edge connects two vertices, so each edge contributes exactly 2 to the total sum of degrees).
*   **Theorem 2:** Handshaking Lemma - Number of odd-degree nodes is always *even*. (*Proof Idea:* $Sum = 2|E|$ which is even. Sum of even degrees is even. For the total to be even, the sum of odd degrees must be even, requiring an even number of odd nodes).
*   **Theorem 3:** In a directed network, Sum of Indegrees = Sum of Outdegrees.

**3. Diagrams in Exam**
*   If asked to define "Diameter", draw a straight line of 5 nodes (A-B-C-D-E). Mark the path from A to E and explicitly state: "Maximum shortest path = 4, therefore Diameter = 4".

**4. Common Mistakes**
*   Calculating density for a directed graph using the undirected formula. (The "2" is only for undirected graphs. For directed, it's just $\frac{|E|}{|V|(|V|-1)}$).
*   Confusing Distance with Diameter. Distance is for a specific pair; Diameter is the max of all distances.

**5. Memory Trick / Mnemonic**
*   **Density Formula:** "Two Edges over V-ictory" $\rightarrow \frac{2E}{V(V-1)}$.

### Topic 4: Centrality Measures (Closeness & Eigenvector)
If SNA is the heart, Centrality is the heartbeat. Evaluators check if you know *why* a measure is used, not just its formula.

**1. Structured Notes & Important Terminology**
*   **Closeness Centrality ($C(v)$):** Detects nodes that can spread information very efficiently. It calculates how close a node is to all other nodes.
*   **Eigenvector Centrality ($x_v$):** Based on the idea that a node is important if it is connected to *other important nodes*.
    *   Formula: $x_v = \frac{1}{\lambda} \sum_{t \in N(v)} x_t = \frac{1}{\lambda} \sum_{t \in V} (a_{vt} \times x_t)$.
    *   $\lambda$ is the largest eigenvalue of adjacency matrix $A$.

**2. High-Weightage Concepts**
*   The matrix equation for Eigenvector: $A \cdot X = \lambda \cdot X$, where $X$ is a column vector of centralities.

**3. Common Mistakes**
*   Missing the $\frac{1}{\lambda}$ factor in the eigenvector formula. Without this normalization factor, the values would just grow to infinity during calculation.

**4. Practical Applications**
*   **Closeness:** Used in epidemic modeling to restrict the spread of disease, or stopping fake news on social media.
*   **Eigenvector:** The foundational concept behind Google's PageRank algorithm.

### Topic 5: PageRank Algorithm
You will absolutely get a numerical on Simple PageRank. Master setting up the linear equations.

**1. Structured Notes & Important Terminology**
*   **Intuition:** A page transfers some of its importance (rank influence) to the pages it links to (forward link). If a highly important node points to you, your status enhances.
*   **Default Rank:** If no hyperlinks exist, every page has uniform rank: $\frac{1}{Number\_of\_Webpages}$.
*   **Simple Ranking Rule:** 
    *   Importance of a webpage = Sum of the importance of its incoming links.
    *   A webpage distributes its importance *equally* to all its outgoing links.

**2. High-Weightage Concepts**
*   Writing the system of linear equations. For example, if Node B points to A and C, it gives $B/2$ to A and $B/2$ to C.

**3. Diagrams in Exam**
*   Whenever you solve a PageRank numerical, draw the node network. Write the rank fraction on the arrows (e.g., if $C$ has 3 outgoing links, write $C/3$ on the arrow pointing to $A$). This secures instant step marks.

**4. Common Mistakes**
*   Dividing a node's rank by its *in-degree* instead of its *out-degree*. Remember: The sender distributes its wealth equally among the people it points to. Always divide by the sender's Outdegree.
*   Forgetting the constraint equation: The sum of all ranks must equal 1 (e.g., $A + B + C = 1$) to solve the final variables.

**5. Memory Trick / Mnemonic**
*   **PageRank Rule:** "Rank Received = Sum of (Sender's Rank / Sender's Out-degree)".

### Comprehensive Glossary of Key Terms
Write these exactly as is if asked for 2-mark definitions:
*   **Adjacency Matrix:** A square $|V| \times |V|$ matrix where $a_{ij}$ indicates an edge between node $i$ and node $j$.
*   **Closeness Centrality:** A metric for detecting nodes that can spread information efficiently through a graph by calculating shortest paths.
*   **Degree Distribution:** The probability $P_k$ that a randomly chosen node in the network has degree $k$.
*   **Diameter:** The maximum distance (shortest path) between any pair of nodes in the network.
*   **Eigenvector Centrality:** A measure where a node's importance is determined by the importance of its neighbors, solved using the largest eigenvalue $\lambda$.
*   **Hypergraph:** A generalized network where a single edge (hyperedge) can connect more than two nodes.
*   **Network Motifs:** Subgraphs that repeat themselves frequently within or across a network at the mesoscopic level.
*   **Signed Network:** A network where edges carry a positive (+) or negative (-) sign representing friendship or enmity.
*   **Strongly Connected:** A directed network where a directed path exists between every single pair of nodes.
